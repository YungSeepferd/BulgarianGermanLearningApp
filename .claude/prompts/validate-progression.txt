# Vocabulary Progression Validation
# Input: vocabulary.json entries
# Output: Detailed progression analysis and learning path optimization

Analyze vocabulary progression across multiple dimensions:

CEFR Alignment:
1. A1 (Breakthrough)
   - Basic everyday expressions
   - Simple personal information
   - Essential survival phrases
   
2. A2 (Waystage)
   - Regular daily activities
   - Simple descriptions
   - Basic social interactions
   
3. B1 (Threshold)
   - Work and study topics
   - Personal opinions
   - Future plans and past experiences
   
4. B2 (Vantage)
   - Technical discussions
   - Abstract concepts
   - Complex opinions

Learning Dependencies:
1. Core vocabulary prerequisites
2. Grammar concept requirements
3. Cultural knowledge needs
4. Communication skill level

Usage Analysis:
1. Frequency in target contexts
2. Pragmatic importance
3. Regional variation impact
4. Register requirements

Validation Rules:
1. CEFR Guidelines
   - Follow official CEFR descriptors
   - Consider both receptive and productive use
   - Align with examination standards
   
2. Learning Dependencies
   - Map prerequisite chains
   - Identify knowledge gaps
   - Note cultural requirements
   
3. Usage Patterns
   - Track frequency data
   - Monitor contextual needs
   - Consider register variation
   
4. Progression Logic
   - Ensure natural learning flow
   - Build on existing knowledge
   - Support skill development

5. Bidirectional Considerations
   - Note BG→DE specific challenges
   - Document DE→BG specific issues
   - Suggest direction-specific learning strategies

DO NOT:
- Ignore official CEFR level descriptors
- Skip prerequisite relationship analysis
- Suggest jumps in difficulty without support
- Mix formal/informal register without notes
- Overlook cultural competency needs
- Break established learning sequences
- Ignore frequency data in level assignment
- Skip validation of suggested changes
- Neglect directional learning differences
- Assume equal difficulty in both directions
